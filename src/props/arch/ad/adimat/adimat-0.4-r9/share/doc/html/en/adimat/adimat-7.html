7. Environment variables

ADiMat uses some enviromnent variables to customize its behaviour. The directory where the additional data is installed may be modified using the ADIMAT_HOME environment variable, for example. If the variables introduced here are not set on the machine, then default values will be used.

Currently these environment variables are used to customize the behaviour of ADiMat:

${ADIMAT_HOME}

Specify the path where the ADiMat-directory-structure is stored. The default value is: '/usr/local'. ADiMat assumes that it's configuration- and additional-files are stored in '${ADIMAT_HOME}/share/adimat'. The binary may reside there two, but it is wise to put it into '${ADIMAT_HOME}/bin'.

${ADIMAT_BUILTINSPATH}

ADiMat scans this path for the '.amb'-files, which are used to define the identifiers build into Matlab. The first file searched is the file 'ADiMat.amb', which includes several more files. This path may be modified by the '.amb'-files (see section builtins for further information). The default path is: '${ADIMAT_HOME}/share/adimat:~/.adimat:.'

$ARCH

Names the machine architecture. This is a string of the pattern <machine cpu type>-<vendor>-<os long name>, where the <machine cpu type> is something like 'i686' or 'm68k', <vendor> is something like 'pc' or 'sun' and <os long name> is something like 'linux-gnu' or 'solaris'. The contents of the $ARCH variable is computed by the config.guess script provided by autoconf. The default for a Pentium III™ pc-system running RedHat Linux 7.2 is: 'i686-pc-linux-gnu'. The default is set at compile time and depends on the hardware and installed software of the system ADiMat is compiled on.

$OSTYPE

Is the <os long name> as defined in the environment variable $ARCH. The default is 'linux-gnu' for a Pentium III™ pc-system running RedHat Linux 7.2. The default is set at compile time and depends on the hardware and installed software of the system ADiMat is compiled on.

$OSNAME

Is the short os name. Examples are 'linux', 'solaris', 'winnt'. The default is set at compile time and depends on the hardware and installed software of the system ADiMat is compiled on.

$VENDOR

Is the name of the vendor of this machine. Examples are 'pc' for personal computers using Intel or AMD processors or 'sun' when running on a system build by Sun. The default is set at compile time and depends on the hardware and installed software of the system ADiMat is compiled on.

Any other environment-variable that is named in a path or likewise, will be replaced by the value the operating system returns when asked for it.
